# 产品原型访谈测试指南

更新时间：2026-05-19

原型文件：[prototype/store-ai-visibility.html](./prototype/store-ai-visibility.html)

## 用途

这不是正式产品界面，而是服务商访谈用的低保真工作台样例。它把“门店 AI 可见性体检”从文字报告变成一个可感知的产品形态，用来测试服务商是否愿意把它放进签约、月报或续费流程。

## 演示顺序

1. 总览
   - 看服务商是否理解 AI 推荐可见性、强推荐率、竞品压制率。
   - 观察他们是否会追问这些指标如何计算。

2. Prompt 结果
   - 看服务商是否认可“用户真实会问这些问题”。
   - 观察他们是否愿意提供自家客户的真实 prompt。

3. 下月任务
   - 重点验证任务是否能进入他们的日常交付。
   - 让他们判断团购标题、短视频、达人 Brief、FAQ 哪个最有价值。

4. 商家月报
   - 让服务商用自己的话复述给商家听。
   - 如果他们能自然复述，说明概念包装有效。

## 需要观察的信号

### 强信号

- 服务商说“这个可以放到我们月报里”。
- 服务商主动问“能不能用我们的客户跑一版”。
- 服务商觉得任务单比报告更有价值。
- 服务商开始讨论定价、批量门店或白标。

### 弱信号

- 服务商只觉得界面好看，但不知道怎么卖。
- 服务商纠结 AI 排名是否稳定，却不关心任务单。
- 服务商认为商家不会理解，也不愿意试讲。
- 服务商不愿提供真实门店。

## 访谈问题

1. 你会把这个工具放在签约前诊断、月报，还是续费沟通里？
2. 这几个指标里，哪个商家最容易听懂？
3. 这几个任务里，哪个最能进入你的真实交付？
4. 如果只保留一个页面，你希望保留哪个？
5. 你愿不愿意提供 1 个真实门店和 3 个竞品，让我们跑一版？
6. 如果每月给 20 家门店生成这样的报告，你觉得合理价格是多少？

## 原型修改优先级

根据访谈反馈优先改：

1. 商家是否听得懂的语言。
2. 服务商是否可直接交付的任务单。
3. 指标是否过于抽象。
4. 报告是否能支持签约/续费。

暂时不要优先改：

- 登录注册。
- 团队权限。
- 复杂趋势图。
- 自动抓取设置。

这些只有在服务商确认愿意试点后才值得做。
