# GEO 赛道进入判断研究

更新时间：2026-05-19

## 目标

判断是否值得投入中国抖音本地生活/实体店服务场景下的 GEO 优化 AI 工具赛道。

目标客户暂定为抖音生活服务运营商/服务商，他们用工具服务线下实体店家。现阶段目标不是立刻做完整 SaaS，而是完成进入判断：市场是否足够、客户是否真痛、国外产品是否可借鉴、技术是否能低成本复制，并形成产品闭环和样例。

## 当前结论 v0.2

谨慎看多，但不建议以“GEO 工具”这个抽象概念直接卖给商家或抖音服务商。更可行的切入是：

> 面向抖音运营商的“门店 AI 可见性体检 + 内容/资料优化任务 + 客户汇报”工具。

也就是把国外 AI visibility/GEO 工具的“监测、诊断、优化、报告”框架，改造成服务商能卖给实体店的交付工具。短期价值不应只承诺 AI 搜索曝光，而应和抖音生活服务现有经营动作绑定：门店资料、短视频/直播脚本、团购卖点、达人 Brief、评价素材、商家月报。

## 进入判断标准

继续投入的条件：

1. 访谈 10 个服务商，至少 4 个愿意拿样例给商家试讲，至少 2 个愿意付费试点。
2. 真实门店测试中，至少 30% 的高意图 prompt 能观察到门店/竞品差异、信息错误或内容缺口。
3. 服务商认为报告可直接用于签约、续费或月报，而不是“有趣但不紧急”。
4. 半自动生成单店月报的人力成本低于服务商可接受价格的 25%。

## 下一步研究缺口

1. 访谈 5-10 个抖音生活服务服务商/运营商，确认他们现在如何向商家卖服务、如何月度汇报、是否需要 AI/搜索相关新卖点。
2. 选择 2-3 个高频行业做样例：餐饮、丽人、酒旅/亲子。
3. 手动跑一批真实门店在豆包、DeepSeek、Kimi、腾讯元宝、百度 AI/文心、夸克等入口的问答可见性，验证“门店级 AI 可见性”是否有明显差异。
4. 用真实门店替换当前虚构样例，形成可演示版本。

## 最小验证动作

推荐直接按 [18_pilot_runbook.md](./18_pilot_runbook.md) 执行。

1. 找 1 家真实餐饮门店和 3 个同城竞品。
2. 用 [tools/prompt-builder.html](./tools/prompt-builder.html) 或 [templates/prompt_bank.csv](./templates/prompt_bank.csv) 生成 20 个 prompts。
3. 在 3 个 AI 入口手动测试，并用 [tools/answer-scorer.html](./tools/answer-scorer.html) 填入回答与评分。
4. 生成一份真实门店体检报告。
5. 拿给 2 个抖音服务商看，记录他们是否愿意试讲、试点或付费。
