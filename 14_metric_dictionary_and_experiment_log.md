# 指标口径与实验记录规范

更新时间：2026-05-19

## 为什么需要统一口径

AI 回答天然会波动。如果指标口径不固定，服务商很难把报告讲清楚，也无法比较复测结果。

本文件用于第一批真实门店测试，确保每家门店按同一套规则记录、评分和解释。

## 核心原则

1. 不把单次 AI 回答当作确定排名。
2. 只把测试结果解释为“可见性信号”。
3. 原始回答必须留档。
4. 指标必须能转化成经营动作。
5. 对商家少讲技术，对服务商保留证据。

## 指标字典

### AI 推荐可见性

定义：

> 目标门店在高意图 prompts 中被提到的比例。

公式：

> 目标门店出现的 prompt 数 / 总 prompt 数

解释：

- 高说明门店更容易被 AI 纳入候选。
- 低说明门店在某些消费场景中缺少可见信号。

注意：

- 只要被提到就算出现，但要区分“出现”和“明确推荐”。

### 强推荐率

定义：

> 目标门店进入前三推荐位，或被 AI 明确表达为推荐选择的比例。

公式：

> 强推荐 prompt 数 / 总 prompt 数

解释：

- 比 AI 推荐可见性更严格。
- 适合给服务商看，不一定适合直接给商家讲。

### 竞品压制率

定义：

> 竞品被推荐但目标门店未被推荐的比例。

公式：

> 竞品出现且目标门店未出现的 prompt 数 / 总 prompt 数

解释：

- 越高说明竞品占据了更多消费场景。
- 是最容易转成销售话术的指标。

### 信息准确性

定义：

> AI 对门店地址、营业时间、价格、人均、特色、预约、服务设施等信息的准确程度。

评分：

- 100：未发现错误。
- 70：有轻微不完整或泛化。
- 40：有明显错误但不影响核心决策。
- 0：关键错误，例如地址、营业时间、价格严重错误。

### 场景覆盖度

定义：

> 门店在目标消费场景中的被推荐和被描述能力。

常见场景：

- 约会
- 生日
- 朋友聚餐
- 亲子
- 游客打卡
- 商务宴请
- 性价比
- 团购/优惠

评分建议：

- 强：多个 AI 入口能稳定推荐，并给出匹配理由。
- 中：偶尔出现，但理由泛化。
- 弱：多数不出现，或被竞品占位。

### 可转任务率

定义：

> 能转化为明确经营任务的 prompt 占比。

公式：

> 能生成任务的 prompt 数 / 总 prompt 数

示例：

- “生日场景未出现” -> 生日服务 FAQ + 生日短视频 + 达人 Brief。
- “抖音团购未被识别” -> 团购标题和卖点改写。
- “价格错误” -> 门店资料和公开页面价格统一。

## Prompt 分级

### 高优先级

- 直接影响消费选择。
- 包含城市、商圈、品类、场景或价格。
- 能转化成抖音经营任务。

示例：

> 杭州湖滨附近适合约会的日料店推荐，环境好一点，人均 200 左右。

### 中优先级

- 能反映场景，但消费意图稍弱。
- 适合补充判断。

### 低优先级

- 过于泛化。
- 不适合第一轮测试。

## 实验记录要求

每次测试必须记录：

- 测试人。
- 测试日期和时间。
- AI 入口。
- 是否登录。
- 是否联网。
- Prompt 原文。
- 原始回答全文。
- 是否出现目标门店。
- 是否出现竞品。
- 疑似错误。
- 截图或导出文件。

## 复测规则

### 7 天复测

用途：

- 判断 AI 回答是否高度波动。
- 不用于证明优化效果。

### 30 天复测

用途：

- 在完成内容/资料/团购/达人任务后观察变化。
- 更适合服务商月报。

### 同批 prompt 复测

必须使用同一批 prompts，否则不可直接比较。

## 报告解释边界

可以说：

- “在这组问题里，竞品更常被推荐。”
- “AI 对门店生日服务信息掌握不足。”
- “这些问题可以转化为下月内容和资料优化任务。”

不要说：

- “我们保证让 AI 推荐你。”
- “这个分数等于真实成交效果。”
- “排名下降说明服务失败。”
- “AI 没推荐就一定没人看到你。”
