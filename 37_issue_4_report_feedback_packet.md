# Issue #4 执行包：用真实报告验证服务商是否觉得有用

更新时间：2026-05-19

对应 GitHub issue：[ #4 Validate report usefulness with service providers](https://github.com/daniel-yu-gold/geo-research/issues/4)

## 目标

拿 Issue #2 生成的真实门店 AI 可见性报告给服务商看，验证它是否真的能进入服务商的签约、月报、续费或增购流程。

这一步要验证的不是“服务商觉得报告好不好看”，而是：

1. 他是否愿意拿给商家试讲。
2. 他会把报告放进哪个业务环节。
3. 他认为哪些内容能促成签约、续费或增购。
4. 他愿意如何付费、试点或转介绍。
5. 他要求下一版报告怎么改，才更容易卖给商家。

## 前置条件

开始 #4 前，至少需要：

1. 1 份真实门店报告，不再使用虚构样例。
2. 报告中每条关键发现都有证据路径。
3. 报告有 200-300 字商家摘要。
4. 已准备服务商内部版备注，能解释证据强弱。
5. 至少 2 个服务商愿意看报告，最好来自 Issue #3 的访谈对象。

如果只有虚构报告，可以做概念访谈，但不能关闭 #4。

## 15 分钟反馈会结构

### 0-2 分钟：设置判断任务

建议说：

> 这是一份真实门店样例，不是完整产品。请你帮我按服务商视角判断三件事：商家听不听得懂、你会不会拿去讲、如果要收费或放进服务包，报告还缺什么。

避免说：

> 你觉得这个 GEO 报告怎么样？

### 2-5 分钟：先让对方自己读

给对方 2-3 分钟自己看，不要急着解释。

观察：

- 他先看分数、发现、任务单，还是原始回答。
- 他有没有主动问证据怎么来的。
- 他能不能复述报告的核心结论。
- 他是否立刻联想到自己的客户。

### 5-10 分钟：逐项追问

必须问：

1. 如果你把这份报告拿给商家，商家最容易听懂哪一页？
2. 哪一页你会删掉或放到内部版？
3. 你会把它用于签约前诊断、月报、续费复盘、投放前策略，还是内容选题？
4. 报告里的任务单能不能进入你们下月交付？
5. 你是否愿意拿这份报告给一个真实商家试讲？

### 10-13 分钟：商业化追问

必须问：

1. 如果每月给 20 家门店出类似报告，你愿意按什么方式付费？
2. 单店报告、服务商账号、门店数、白标月报、商家增值包分成，哪个更合理？
3. 什么价格你会觉得可以试？
4. 什么结果出现，你愿意连续付费 3 个月？
5. 如果你不愿意付费，原因是什么？

### 13-15 分钟：争取具体下一步

强推进：

> 你愿意拿这份报告给一个商家试讲吗？如果商家有反馈，我们一起改下一版。

中推进：

> 你愿意给我另一个真实门店，让我们按这个格式再跑一版吗？

弱推进：

> 你能帮我标注一下哪些内容商家看不懂、哪些内容你不会放进月报吗？

## 必须记录的字段

反馈结果记录到 [templates/report_feedback.csv](./templates/report_feedback.csv)，核心字段包括：

- 服务商信息
- 报告版本
- 是否能听懂
- 最有价值页面
- 最没用页面
- 最适合使用场景
- 是否愿意给商家试讲
- 是否愿意提供下一家真实门店
- 是否愿意付费试点
- 可接受价格
- 必改项
- 关键原话
- 信号等级

关键证据同步写入 [templates/evidence_index.csv](./templates/evidence_index.csv)。

## 判级规则

| 等级 | 判定标准 | 下一步 |
| --- | --- | --- |
| A | 愿意拿报告给商家试讲，或愿意提供下一家真实门店继续跑 | 进入付费试点设计，更新 MVP |
| B | 认可月报/续费用途，但需要改报告语言或格式 | 按反馈改报告，再二访 |
| C | 觉得有趣但不愿试讲、不愿给门店、不谈价格 | 作为弱信号，暂不投入开发 |
| D | 明确认为商家不会买、和现有工具重复、无法进入交付 | 写入 No-Go 证据 |

## 报告改版规则

根据反馈改版时，优先级如下：

1. 商家是否能听懂。
2. 服务商能否直接拿去讲。
3. 任务单是否能进入真实交付。
4. 证据是否足够让服务商放心。
5. 指标是否能支持月报或续费。

暂时不要优先做：

- 更复杂的可视化。
- 自动化采集。
- 多账号权限。
- 大量趋势图。
- 细颗粒模型评分。

这些只有在 A/B 信号足够多时才值得做。

## 2 个反馈后的快速判断

Issue #4 最少需要 2 个服务商反馈。完成 2 个后先做一次小判断：

### 继续推进信号

满足任意 2 条：

1. 至少 1 个愿意给商家试讲。
2. 至少 1 个愿意提供下一家真实门店。
3. 至少 1 个愿意讨论付费试点。
4. 2 个都认为任务单比单纯报告有价值。
5. 2 个都认为报告能进入月报、签约或续费流程。

### 暂缓信号

出现任意 2 条：

1. 两个服务商都认为商家听不懂。
2. 两个服务商都不愿拿给商家看。
3. 两个服务商都只愿意免费附赠，不愿作为增值模块或工具付费。
4. 两个服务商都认为和现有工具重复。
5. 两个服务商都无法说出具体使用场景。

## 关闭标准

Issue #4 可以关闭的最低标准：

1. 至少 2 个服务商看过真实报告。
2. 每个反馈都记录在 [templates/report_feedback.csv](./templates/report_feedback.csv)。
3. 关键原话、会议纪要或截图登记到 [templates/evidence_index.csv](./templates/evidence_index.csv)。
4. 明确判断报告是否能进入签约、月报、续费或增购流程。
5. 明确下一版报告要保留、删除或强化哪些内容。
