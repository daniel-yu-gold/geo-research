# 技术可实现性与 MVP 架构

更新时间：2026-05-19

## 结论

技术上可行，但不要一开始追求全自动、全平台、实时监控。

推荐路线是：

1. 先做半自动“体检报告生成器”，验证服务商是否愿意用。
2. 再自动化 prompt 生成、结果录入、指标计算和报告生成。
3. 最后再逐步接入更稳定的数据源和平台自动采集。

核心难点不是 LLM 分析，而是国内多平台数据获取、AI 入口自动化稳定性、以及报告能否和抖音服务商的交付流程绑定。

## MVP 范围

### 必做

- 门店资料录入。
- 竞品录入。
- 行业/城市/场景 prompt 生成。
- AI 回答结果录入：早期可以人工粘贴。
- 品牌/竞品提及抽取。
- 推荐位置、推荐理由、信息错误、竞品差距分析。
- 优化任务生成。
- 服务商客户报告导出。

### 暂不做

- 全平台自动爬取。
- 实时排名波动监控。
- 自动归因到 GMV。
- 抖音官方数据深度接入。
- 商家自助购买和复杂权限体系。

### Competitor

- 地址/商圈
- 链接
- 差异化标签

### Prompt

- prompt_id
- store_id

### AnswerRun

## 核心指标

### 门店层

- AI 推荐覆盖率：目标门店在多少高意图 prompt 中被提及。
- 强推荐率：目标门店是否进入前 3 个推荐位或被明确推荐。
- 竞品压制率：竞品出现但目标门店未出现的比例。
- 信息准确率：地址、营业时间、价格、人均、特色是否准确。
- 场景覆盖度：约会/亲子/聚餐/游客/生日等场景覆盖情况。

### 来源层

- 来源提及率：回答是否引用或提及抖音、地图、点评、小红书、官网、媒体等。
- 来源缺口：目标门店在哪些来源明显弱于竞品。
- 内容资产缺口：哪些 prompt 缺少对应内容资产。

### 服务商层

- 月报生成时间。
- 每店任务数。
- 已完成任务数。
- 可用于客户沟通的亮点数。

## 分析流程

1. 生成 prompts
   - 输入门店、品类、城市、商圈和目标场景。
   - 用模板生成 20-50 个高意图问题。

2. 收集回答
   - MVP 阶段可人工在多个 AI 入口提问并粘贴回答。
   - 保存原文和时间，避免只保存摘要。

3. 结构化解析
   - 抽取目标门店是否出现。
   - 抽取竞品是否出现。
   - 抽取推荐理由、场景、人均、地址、营业时间。
   - 标记疑似错误。

4. 评分
   - 按 prompt 优先级加权。
   - 本地消费决策类 prompt 权重高于泛泛品牌词。

5. 生成任务
   - 每个问题映射到具体交付物。
   - 示例：生日场景没出现 -> 生日场景短视频脚本 + 团购标题优化 + 达人 Brief。

6. 输出报告
   - 给服务商看的经营视角。
   - 给商家看的非技术语言。

## 平台自动化可行性

### 低风险

- 手动录入 AI 回答。
- 使用公开网页/搜索结果做辅助验证。
- 使用服务商提供的门店资料和内容链接。
- 自动生成 prompts、诊断、报告和任务。

### 中风险

- 自动化访问 AI 网页入口。
- 批量抓取公开搜索结果。
- 根据小红书/点评/地图公开页面做结构化分析。

### 高风险

- 大规模自动化调用非公开接口。
- 绕过平台限制。
- 声称可以稳定影响 AI 推荐排名。
- 把 AI 可见性和抖音成交做强归因。

## 推荐技术路线

### v0：手工样例

- Markdown/表格记录门店、prompt、回答。
- 人工跑 AI 入口。
- 用 LLM 辅助生成诊断和报告。

### v1：内部 Web 工具

- 门店和竞品录入。
- prompt 自动生成。
- 回答粘贴和结构化。
- 报告一键生成。

### v2：服务商工作台

- 多门店管理。
- 行业模板。
- 月报批量生成。
- 任务看板。
- 白标导出。

### v3：自动监控

- 稳定接入部分 AI/搜索入口。
- 周期复测。
- 趋势图和竞品变化。
- API 或自动化队列。

## 技术投入估算

### v0

- 1 人，3-5 天。
- 目标：做出能访谈演示的真实样例。

### v1

- 1-2 人，2-4 周。
- 目标：支持 20-50 家门店的半自动试点。

### v2

- 2-4 人，2-3 个月。
- 目标：支持服务商多客户交付。

## 关键验证

先验证 3 件事：

1. AI 回答里是否真的存在可观察的门店/竞品差异。
2. 服务商是否认可这些差异能转化成内容和经营任务。
3. 半自动生成报告的人力成本是否低于客户愿意支付价格。

如果这 3 件事成立，再做自动化才值得。
